Updating OS and Software

Updating the OS

Updating and upgrading the operating system, and cleaning up unnecessary files:

# Updating the OS
sudo apt update && sudo apt upgrade -y && sudo apt autoremove && sudo apt autoclean

Adding an Alias to Your Bash Profile

Create a shortcut command for updating the system:

  1. Open your Bash profile in nano: 
    sudo nano ~/.bashrc
  2. Add this line to the end of the file: 
    alias update='sudo apt update && sudo apt upgrade -y && sudo apt autoremove && sudo apt autoclean'
  3. Save and close the file, then apply the changes: 
    source ~/.bashrc

NVM & Git Software

Installing Needed Tools

Install essential tools and packages:

sudo apt install -y build-essential wget curl gnupg
sudo apt install -y python3 python3-pip
sudo apt install -y maven

Installing GIT with current Release

Install Git:

# For Ubuntu, this PPA provides the latest stable upstream Git version
sudo add-apt-repository ppa:git-core/ppa 
sudo apt update; 
sudo apt install git

Installing NVM Packages

Install NVM (Node Version Manager):

curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash

Reset bash after install:

source ~/.bashrc

Then install the latest LTS version of Node.js:

nvm install --lts

sudo snap install code --classic

Install Postman using Snap:

Install Zoom using Snap:

Optional Install Obsidian using Snap: (Note Taking App)

sudo snap install obsidian

Installing Chrome-Browser

Download Google Chrome Package

Download Chrome package:

wget https://dl.google.com/linux/direct/google-chrome-stable_current_amd64.deb

Install Chrome Package

Install and download Chrome:

sudo dpkg -i google-chrome-stable_current_amd64.deb

Fix any Missing Dependencies

Resolve any missing dependencies:

sudo apt install -f

Reconfigure any packages if necessary:

sudo dpkg --configure -a

Docker Setup

Add Docker's Official GPG Key

Add Docker's GPG key:

sudo apt update
sudo apt install ca-certificates
sudo install -m 0755 -d /etc/apt/keyrings
sudo curl -fsSL https://download.docker.com/linux/ubuntu/gpg -o /etc/apt/keyrings/docker.asc
sudo chmod a+r /etc/apt/keyrings/docker.asc

Add Docker Repository to APT Sources

Add the Docker repository:

echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.asc] https://download.docker.com/linux/ubuntu \
  $(. /etc/os-release && echo "$VERSION_CODENAME") stable" | \
  sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

Install Docker Engine

Update package list and install Docker:

sudo apt update
sudo apt install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

Test Docker Installation

Run a test Docker container:

sudo docker run hello-world

Setup Docker to Run Without sudo

Configure Docker to run without sudo:

  1. Create the Docker group (if not already created): 
    sudo groupadd docker
  2. Add your user to the Docker group (replace coder with your username): 
    sudo usermod -aG docker coder
  3. Apply the new group membership: 
    newgrp docker
  4. Verify the setup: 
    docker run hello-world

Docker Script

Setting up a Script to Cleanup Docker

  1. Create the Shell Script: 
    nano cleanup.sh
  2. Write your script and save the file: 
    #!/bin/bash
sudo docker stop $(sudo docker ps -aq)
sudo docker rm $(sudo docker ps -aq)
sudo docker rmi $(sudo docker images -q)
sudo docker volume rm $(sudo docker volume ls -q)
sudo docker network rm $(sudo docker network ls -q)
  3. Make the Script Executable: 
    chmod +x cleanup.sh
  4. Run the Script: 
    sudo ./cleanup.sh

You can have this saved in any directory and use it.

Setting Up Git Global Configuration

Step 1: Set Up Git Global Configurations

Set your Git global username and email:

git config --global user.name "Gitlab User Name"
git config --global user.email "Gitlab Email"

Step 2: Configure Git to Sign Commits with GPG

Generate a GPG Key

  1. Generate a new GPG key: 
    gpg --full-generate-key
    • Select the key type (RSA and RSA recommended).
    • Choose the key size (4096 bits recommended).
    • Set the key expiration.
    • Provide your name and email when prompted.
    • Set a passphrase for your GPG key.
  2. List your GPG keys to find the key ID: 
    gpg --list-secret-keys --keyid-format LONG

    Look for the line starting with sec and note the key ID (a 16-character hexadecimal string).

  3. Copy your GPG key to the clipboard (Ctrl+Shift+C): 
    gpg --armor --export your_key_id
  4. Add the GPG key to your GitLab account.

Configure Git to Use Your GPG Key

  1. Set the signing key in Git: 
    git config --global user.signingkey your_key_id
  2. Enable GPG signing for all commits: 
    git config --global commit.gpgsign true
  3. Disable SSL verification: 
    git config --global http.sslVerify false
  4. Storing GPG credentials: 
    git config --global credential.helper store

Step 3: Verify Your Configuration

To see all your global configurations:

git config --global --list

VPN Setup

Downloading the VPN Configuration File

  1. Go to the VPN page.
  2. Click on the VPN button.
  3. You will get a pop-up, Click the Start button.
  4. Another pop-up will show and then Click the Check Box and then Click Open xdg-open.
  5. Click More options.
  6. Click Install/Reinstall Network Access Application.
  7. Click 64 bit .deb (Ubuntu/Debian), then Click Download.
  8. You will download a file called linux_f5vpn.x86_64.deb.
  9. Go to your Downloads directory: 
    cd ~/Downloads
sudo dpkg -i linux_f5vpn.x86_64.deb
  10. Once that is installed go back to the browser and Click on Click here.
  11. Then Click Start.
  12. Then follow the prompts and you're ready to go for the VPN access.
